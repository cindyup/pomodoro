# 番茄钟 · Pomodoro Timer

一款轻量、零依赖的番茄钟应用，支持任务管理、专注记录、周报看板与自动分类总结。

## 产品定位

面向个人开发者与知识工作者的轻量级番茄钟工具。无需注册、无需安装依赖、无需外部服务，双击即用。专注记录和工作数据完全存储在本地。

### 核心原则

| 原则 | 说明 |
|---|---|
| **零依赖** | 不依赖任何第三方库或框架，纯 HTML/CSS/JS + Node.js 标准库 |
| **隐私优先** | 所有数据存储在本机，无云端上传 |
| **离线可用** | 浏览器打开即可使用，服务器仅用于数据持久化和周报 API |
| **单文件** | 前端为单 HTML 文件，清晰可读 |

## 功能清单

### P0 · 核心功能

| 功能 | 描述 |
|---|---|
| **番茄计时** | 25 分钟专注计时，支持开始/暂停/重置/提前完成 |
| **三种模式** | 专注模式（25min）、短休息（5min）、长休息（15min），时长均可自定义 |
| **任务管理** | 添加待办任务、勾选完成、删除任务、点击任务自动开始专注 |
| **专注记录** | 每个任务的番茄数和专注时长自动统计，页面展示 |
| **数据持久化** | 通过本地 Node.js 服务将数据持久化到 JSON 文件 |
| **端口自动切换** | 检测端口占用并自动切换到下一个可用端口 |

### P1 · 周报看板

| 功能 | 描述 |
|---|---|
| **每周总结** | 查看本周已完成和待滚动的任务，含专注时间统计 |
| **柱状图可视化** | 每个任务以横向柱状图展示专注时长，比例直观 |
| **上周归档** | 可折叠展示上周完成/未完成的任务列表 |
| **周五自动打开** | 周五首次打开页面时自动弹出周报 |
| **侧边栏入口** | 任何时候可点击左侧「周报」侧边栏手动打开 |

### P2 · 智能分类总结

| 功能 | 描述 |
|---|---|
| **自动分类** | 根据任务内容关键词自动归入「研发类」「会议类」「办公材料类」 |
| **分类看板** | 本周各分类的专注时长卡片，高亮占比最大的类别 |
| **AI 风格总结** | 自动生成一句话总结（精力分布 + 夸奖 + 建议） |
| **手动修正** | 未分类的任务可在周报中手动选择分类，即时刷新分析 |

### P3 · 体验增强

| 功能 | 描述 |
|---|---|
| **隐藏已完成** | 主页面已完成任务默认折叠，保持界面清爽 |
| **通知提醒** | 专注/休息结束时浏览器通知 + 声音提示 |
| **SVG 进度环** | 计时器圆形进度动画，实时反馈 |

## 技术架构

```
┌──────────────────────────────────────────────────┐
│                   Browser                         │
│  ┌─────────────────────────────────────────────┐  │
│  │         pomodoro.html (单文件)               │  │
│  │  ┌─────────┐ ┌──────────┐ ┌─────────────┐  │  │
│  │  │ 计时器   │ │ 任务管理  │ │  周报看板    │  │  │
│  │  │ 核心     │ │ CRUD     │ │ 图表+分类    │  │  │
│  │  └─────────┘ └──────────┘ └─────────────┘  │  │
│  └─────────────────────────────────────────────┘  │
│                    ↕ fetch/API                     │
└────────────────────┬─────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────┐
│            pomodoro.js (Node.js 服务器)            │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 静态文件  │ │ REST API │ │  周报计算引擎     │  │
│  │ 服务      │ │ CRUD     │ │  分类+统计+总结   │  │
│  └──────────┘ └──────────┘ └──────────────────┘  │
└────────────────────┬─────────────────────────────┘
                     │
        ┌────────────▼────────────┐
        │  pomodoro-data.json     │
        │  本地持久化存储          │
        └─────────────────────────┘
```

### 前端（pomodoro.html）

- **框架**: 无（原生 Vanilla JS）
- **状态管理**: 全局 `state` 对象，`localStorage` 降级存储
- **计时器**: `setInterval` 驱动，SVG `stroke-dashoffset` 进度环
- **通信**: `fetch` 调用后端 API，服务不可用时静默降级到 `localStorage`
- **周报**: 动态 DOM 渲染柱状图，CSS transition 动画

### 后端（pomodoro.js）

- **运行时**: Node.js 标准库（http、fs、path）
- **端口策略**: 从 3457 开始检测，被占用则 +1 递增尝试，最多 10 次
- **API**: 5 个 REST 端点（加载/保存/导入/周报/分类）
- **分类引擎**: 基于关键词匹配的自动分类规则，支持手动覆盖

## 快速开始

### 前置条件

- [Node.js](https://nodejs.org/)（任何版本，建议 v16+）
- 现代浏览器（Chrome / Edge / Firefox）

### 启动方式

**方式一：双击启动（推荐）**

双击 `启动番茄钟.bat`，自动打开浏览器。

**方式二：命令行启动**

```bash
node pomodoro.js
```

服务启动后自动打开 `http://localhost:3457`。如果端口被占用，会自动尝试后续端口（3458、3459 …）。

**方式三：直接打开 HTML（纯前端模式）**

直接用浏览器打开 `pomodoro.html`，数据会保存到浏览器 `localStorage`。部分功能（周报分析、分类总结）需要后端服务。

## 使用指南

### 基础操作

1. **添加任务** — 底部输入框输入任务描述，按回车或点「+」
2. **开始专注** — 点击任务选中，计时器自动开始（或点「开始」按钮）
3. **完成番茄** — 计时归零自动记录，或点「完成」提前结束
4. **休息** — 完成一个番茄后自动切换到休息模式
5. **完成任务** — 勾选任务前的复选框

### 周报看板

1. 点击左侧红色「周报」侧边栏打开
2. 查看本周已完成/待滚动的任务柱状图
3. 点击「📅 上周完成的任务」展开归档
4. 未分类的任务会显示分类选择按钮，点击即可修正

### 分类说明

| 分类 | 自动匹配关键词 | 手动设置 |
|---|---|---|
| 研发类 | 开发、部署、代码、重构、bug、调试、测试、前端、后端、API、数据库、优化、PR、Review、合并、分支、需求、功能、升级、迁移、配置、分析 | 周报中点击对应按钮 |
| 会议类 | 会议、同步、评审、复盘、讨论、对齐、宣讲、培训、晨会、周会、访谈、1on1、Presentation | 周报中点击对应按钮 |
| 办公材料类 | 文档、材料、邮件、PPT、方案、报告、复盘、Wiki、周报、规范、流程、申请、报销、审批、填写、整理 | 周报中点击对应按钮 |
| 未分类 | 无关键词匹配 | 手动选择分类 |

## 数据存储

### 数据结构（pomodoro-data.json）

```json
{
  "completed": 42,
  "sessionInRound": 2,
  "currentTaskIndex": 1,
  "tasks": [
    {
      "text": "实现用户登录功能",
      "done": true,
      "category": "dev",
      "sessions": [
        { "timestamp": 1715123456789, "duration": 1500 }
      ],
      "createdAt": 1715123000000,
      "completedAt": 1715125000000
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `completed` | number | 总完成番茄数 |
| `sessionInRound` | number | 当前轮次（每 4 个一轮长休息） |
| `currentTaskIndex` | number | 当前选中的任务索引 |
| `tasks[]` | array | 任务列表 |
| `tasks[].text` | string | 任务描述 |
| `tasks[].done` | boolean | 是否完成 |
| `tasks[].category` | string | 分类（dev/meeting/doc/uncategorized） |
| `tasks[].sessions[]` | array | 专注记录列表 |
| `tasks[].sessions[].timestamp` | number | 专注完成时间戳 |
| `tasks[].sessions[].duration` | number | 本次专注时长（秒） |
| `tasks[].createdAt` | number | 任务创建时间戳 |
| `tasks[].completedAt` | number | 任务完成时间戳 |

### 降级策略

服务器不可用时，自动降级到浏览器 `localStorage`（key: `pomodoro-state`），恢复服务后自动切换回服务器存储。

## API 文档

| 端点 | 方法 | 描述 |
|---|---|---|
| `/api/load` | GET | 加载所有数据 |
| `/api/save` | POST | 保存所有数据 |
| `/api/import` | POST | 导入数据（form 表单，兼容 file:// 协议） |
| `/api/weekly-summary` | GET | 本周+上周总结（含已完成/未完成） |
| `/api/weekly-analysis` | GET | 本周分类分析（含各分类统计和 AI 总结） |
| `/api/set-category` | POST | 设置任务分类 |
